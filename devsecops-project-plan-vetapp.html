AI Terraform analyzer

A unique project feature — an AI-powered tool that reviews your infrastructure on every deploy.

Terraform AI Review Bot

A Python script that runs in your CD pipeline. It reads the current Terraform plan, sends it to Claude API, and posts an analysis as a GitHub PR comment or pipeline summary.

terraform plan -json
→
Parse & extract changes
→
Send to Claude API
→
Post analysis to GitHub

What the AI analyzes:

• Security review — open ports, public access, missing encryption, overly permissive IAM
• Cost estimation — what resources are being added and approximate monthly cost
• Best practices — resource naming, tagging, redundancy, backup policies
• Change summary — human-readable summary of what will change in infrastructure
• Risk assessment — flags destructive changes (deleting resources, changing security groups)
# scripts/terraform_ai_review.py (simplified) import subprocess, json import anthropic # 1. Get terraform plan plan_json = subprocess.check_output(["terraform", "show", "-json", "tfplan"]) parsed = json.loads(plan_json) # 2. Extract resource changes changes = [ {"action": r["change"]["actions"], "type": r["type"], "name": r["name"]} for r in parsed.get("resource_changes", []) ] # 3. Send to Claude for analysis client = anthropic.Anthropic() response = client.messages.create( model="claude-sonnet-4-6", max_tokens=1000, messages=[{ "role": "user", "content": f"""Analyze this Terraform plan for a vet clinic app (TeyaVet). Flag security issues, estimate costs, and summarize changes. Plan: {json.dumps(changes, indent=2)}""" }] ) # 4. Post as GitHub comment or output to pipeline print(response.content[0].text)
🎯 This is your differentiator. No other team will have an AI-powered infrastructure review. During the demo, show a Terraform PR where the AI catches a security issue (like an open port 22 or missing MySQL encryption) and recommends a fix.
